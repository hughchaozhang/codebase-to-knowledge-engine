The product in one sentence

Autoresearch is a tiny research lab in a repo: an AI agent edits one training file, runs a five-minute experiment, checks the score, and decides what to try next.

🧠
program.md

The human-written mission brief for the agent.

🛠️
train.py

The only file the agent is expected to change during experiments.

📦
prepare.py

Downloads data, trains the tokenizer, and exposes runtime utilities.

💡
Why you should care

If you understand which file sets the rules and which file the agent mutates, you can steer an AI researcher without getting buried in ML internals.

Think of it like a racing pit crew

A driver comes in, the crew swaps one part, sends the car out for a short lap, and reads the stopwatch. Autoresearch works the same way: make one change, run a short test, keep the good ideas, throw out the bad ones.

The project’s whole power move fits in three bullets

🎯

One editable file

The agent mainly changes train.py, which keeps diffs reviewable and the search space small.

⏱️

Fixed time budget

Every run gets the same five-minute wall-clock budget, so experiments are easier to compare fairly.

📉

One metric

val_bpb decides whether a change helped or hurt.

Code ↔ English: the fixed time budget

The repo bakes the five-minute limit into the prep layer so every experiment plays by the same clock.

CODE 

TIME_BUDGET = 300        # training time budget in seconds (5 minutes)
EVAL_TOKENS = 40 * 524288  # number of tokens for val eval
PLAIN ENGLISH

Every training run gets 300 seconds — exactly five minutes — so no experiment gets extra time to look artificially better.

The evaluation also uses a fixed amount of validation text, which makes the score more comparable from run to run.

Code ↔ English: one file is fair game

This sentence in the README quietly tells you who controls what.

CODE 

- **`prepare.py`** — constants, one-time data prep, and runtime utilities. Not modified.
- **`train.py`** — the single file the agent edits.
- **`program.md`** — baseline instructions for one agent.
PLAIN ENGLISH

Leave the prep layer alone unless you intentionally want to change the rules of the experiment.

Let the agent mutate train.py — that is the sandbox where model ideas get tested.

Humans mostly edit program.md, which is more like managing the research team than writing the training code yourself.

Architecture lesson

Good systems create clean boundaries: one place for human intent, one place for machine experimentation, one place for fixed infrastructure.
